# v0.1.1 Release Checklist

Use this checklist for the first patch tag after the published `v0.1.0` baseline.

## Scope

`v0.1.1` should package the hardening that already landed on `main` after the
older `v0.1.0` tag. It should not introduce new skills during the tagging pass.

Included readiness changes:

- local Markdown link validation,
- required `README.md` files for every skill package,
- maintainer PR template,
- security reporting policy,
- context budget skill,
- Windows installation guide,
- maintainer readiness report,
- CI readiness badge,
- first-release checklist.

## Preconditions

- `python scripts/validate-skills.py` passes locally.
- The public `validate` workflow is green on the exact commit to tag.
- `CHANGELOG.md` has a `v0.1.1` entry summarizing only post-`v0.1.0` changes.
- `docs/post-v0.1.0-audit-2026-07-13.md` still matches the public tag target.
- No skill examples or docs include private repository names, access tokens,
  customer data, or unpublished vulnerability details.

## Release steps

1. Run `python scripts/validate-skills.py`.
2. Confirm `git status --short --branch` is clean and aligned with `origin/main`.
3. Confirm the GitHub Actions badge points to a successful `validate` run.
4. Move the relevant `Unreleased` changelog notes under `v0.1.1`.
5. Create an annotated tag:

   ```powershell
   git tag -a v0.1.1 -m "codex-skills-cn v0.1.1"
   git push origin v0.1.1
   ```

6. Check the public tag page renders README links, skill paths, and docs links.

## Do not release if

- Any validation check fails locally or in GitHub Actions.
- The changelog implies that `v0.1.1` contains new skills added during the tag
  pass.
- The checklist is being used to rewrite or move the existing `v0.1.0` tag.
