# 贡献指南

欢迎提交新的 Codex skill、中文适配、使用案例和修正。

## 提交前请确认

- 这个 skill 有明确用途，不是泛泛而谈的提示词。
- 内容适合 Codex 使用，而不是只适合其他 agent 的专有命令。
- `SKILL.md` 包含触发场景、工作流、边界和验证方式。
- 如果来自第三方项目，请注明来源和许可证。

## 推荐流程

1. Fork 本仓库。
2. 在 `skills/<skill-name>/` 下新增或修改内容。
3. 按 `docs/skill-template.md` 补齐说明。
4. 运行 `python scripts/validate-skills.py`。
5. 提交 Pull Request，并说明适用场景和测试方式。

## PR 说明建议

请尽量包含：

- 这个 skill 解决什么真实问题。
- 适合哪些用户或维护场景。
- 不适合哪些场景。
- 你如何验证它不会误导 Codex。
- 是否引用或改写了第三方内容，以及来源和许可证。

## 命名规范

- 目录名使用英文小写和连字符，例如 `web-extraction`。
- Skill 名称尽量短，能说明用途。
- 中文说明要直接、可执行，少写口号。

## 不接受的内容

- 要求用户输入密钥但没有安全说明的 skill。
- 鼓励绕过付费墙、验证码、登录限制或网站规则的 skill。
- 没有真实使用场景的空泛提示词集合。
- 大量复制第三方内容且没有来源说明的提交。

## 质量标准

一个可合并的 skill 至少应该做到：

- frontmatter 中 `name` 与目录名一致。
- `description` 清楚说明何时触发，而不是只解释它是什么。
- 包含适用场景、不适用场景、工作流和验证方式。
- 对外部工具、网络访问、数据采集、账号权限等场景写明安全边界。
- 语言简洁，默认 Codex 已经很聪明，只补它做任务时真正需要的流程知识。
